# CRUD_Flashcard
Aplicação de CRUD, manipulação de arrays e modularização em JavaScript para construir um Sistema de Gerenciamento de Flashcards interativo no terminal.

# Dicas pra usar o Github
## Criar uma branch (ESSENCIAL, USAR NO TERMINAL)
- clonar repositório: git clone https://github.com/AmirMzKT/CRUD_Flashcard
- comando: git checkout -b feature-cadastro
- cada pessoa cria a sua (ex: feature-login, feature-listagem)

# Depois de codar (nessa ordem)
1. git add . (sempre verificar se está dentro da pasta)
2. git commit -m "feat: adiciona função tal"
3. git push origin feature-funcao

# No Github
- Procure por **Compare & Pull Request**
- Explica o que fez. Ex: ## O que foi feito - Função do flashcard
- Envia

# Merge (Revisar e Juntar)
- Quem faz isso é o dono do repositório, pórem, é interessante todo mundo revisar o que fez

# Regrinhas
## Não fazer
- Não mexer direto no **main**
- Não subir código quebrado (sempre verificar antes de subir e/ou perguntar se estiver com dúvidas)
- Não sobrescrever código dos outros (pode dar conflitos ou confusão na hora de usar o código completo)
## Boas Práticas
- Commits claros: feat: adiciona função ou fix: corrige erro tal
- Sempre atualizar antes: git pull origin main
