# Dicas pra usar o Github
## Criar uma branch (ESSENCIAL, USAR NO TERMINAL)
- clonar repositório: git clone https://github.com/AmirMzKT/CRUD_Flashcard
- comando para criar branch: git checkout -b feature-nome
- cada um cria a sua branch de acordo com o que está fazendo (ex: feature-flashcard, feature-funcao)
---
# Depois de codar
1. git add . (sempre verificar se está dentro da pasta)
2. git commit -m "feat: adiciona função tal"
3. git push origin feature-exemplo
4. No Github, procure por **Compare & Pull Request**
5. Explica o que fez. Ex: ## O que foi feito - Função do flashcard, depois é só enviar
6. Você pode usar a extensão do Github no VS Code, é mais simples e prático, vai de sua preferência
---
## Usando a extensão no VS Code
1. Antes, verifique se você está na main atualizada (git pull origin main)
2. Abra **Controle de Código-Fonte** (Ctrl+Shift+G)
3. Clique nos três pontos (Icone: **...**) ao lado de **Alterações** e procure por **Branch**
4. Em **Branch**, clique em **Criar ramificação**, dê o nome (ex: feature-nome), depois volte na mesma aba e clique **Publicar ramificação**
5. Depois clique em **Efetuar o Pull** 
- Sempre lembre de verificar se está na branch correta, pra evitar problemas no **main**

# Merge (Revisar e Juntar)
- Quem faz isso é o dono do repositório, pórem, é interessante todo mundo revisar o que cada um fez, pode mandar print/foto ou o proprio arquivo do código fora do Github pra todos revisarem
---
# Regrinhas
## Não fazer
- Não mexer direto no **main**
- Não subir código quebrado (sempre verificar antes de subir e/ou perguntar se estiver com dúvidas)
- Não sobrescrever código dos outros (pode dar conflitos ou confusão na hora de usar o código completo)
- Não subir o código na branch errada
## Boas Práticas
- Código legível, isso facilita pra todo mundo entender o que está sendo feito 
- Sempre teste seu código antes de enviar para o Github
- Sempre atualizar antes de iniciar: git pull origin main
- Verificar sempre a branch
- Commits claros: feat: adiciona função ou fix: corrige erro tal
---
# Fluxo de trabalho
1. Atualizar projeto localmente: git pull origin main
2. Criar branch: git checkout -b feature-nome
3. Codar: Faça sua parte do projeto e sempre testar o código antes de mandar
4. Conferir branch: veja se está na branch correta
5. Adicionar alterações: git add .
6. Dar commit: git commit -m "feat: alteração feita"
7. Enviar (push): git push origin feature-nome ou na extensão **"Publicar ramificação"**
8. Criar Pull Request (PR): No Github, procure por **"Create Pull Request"** ou no VS Code usando a extensão e depois explicar o que foi feito
9. Revisão do código
10. Merge: Aprovar e juntar na **main** (quem faz isso é o dono do repositório)
11. Atualizar novamente (Primeira linha)
